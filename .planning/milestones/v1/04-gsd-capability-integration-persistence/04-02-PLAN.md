<objective>
Build the execute gate: a thin hook that RELOADS the persisted selection state (rather than re-deriving the signal), re-checks budget, renders the `<governance>` fragment into the otherwise-empty executor/subagent context, and the reload-after-boundary integration test that proves governance state survives a compaction/subagent boundary (the core ENF-01 acceptance evidence, criterion 4).

This is the project's proof that "governance at scale over a long runtime" works.

Thin per Pitfall 3: locate `.planning/governance/selection-state.json` (it MUST already exist from a prior discuss:pre — if it does not, fail loud), invoke `executeHook` (`node dist/governance/execute-hook.js`), and let the hook render + write the `<governance>` fragment to the produces artifact path the loop picks up for the executor/subagent context. The SKILL does NOT re-run selection, classify risk, or read STATE — discuss already did that once; execute reloads.
- `src/governance/execute-hook.ts` — the thin execute hook adapter. Exports `executeHook(args: { projectRoot: string; statePath?: string }): { fragment: string; record: GovernanceRecord }`. Reads the persisted record via `readSelection(args.statePath ?? selectionStatePath(args.projectRoot))` — fail loud if missing/malformed (never silent empty fragment — Pitfall 7). Calls `renderInjection(record.selectionResult)` to render the fragment (single source of truth for rendering). Re-checks `record.selectionResult.budgetExceeded` and, if true, writes a stderr warning naming `budget.used`, `budget.limit`, `budget.offenders` and sets `process.exitCode = 1` AFTER emitting the fragment (mirrors `src/cli/commands/inject.ts` — CR-02: process.exitCode not process.exit, fragment always observable). Returns `{ fragment, record }` so the SKILL/host can attach the fragment. Does NOT re-derive the signal, NOT call select, NOT call classifyRisk (D-RELOAD — Pitfall 6).
- `src/governance/execute-hook.test.ts` — RED-first node:test suite over a temp project root: (a) happy path — writeSelection a record, then call executeHook with NO signal in memory, and assert the returned fragment equals `renderInjection(record.selectionResult)`; (b) reload-not-rederive — writeSelection a record, then mutate the in-memory index/state files (or pass a different indexPath that doesn't exist) and assert executeHook STILL returns the original fragment (proves it reads the persisted selectionResult, not re-derives); (c) budget continuity — writeSelection a record whose `selectionResult.budgetExceeded = true`, call executeHook, assert non-zero exit + stderr warning + fragment still emitted; (d) loud-on-missing — no selection-state.json at the path -> executeHook throws; (e) loud-on-malformed — writeFileSync a non-JSON payload -> executeHook throws (readSelection propagates).
- `src/governance/reload-boundary.test.ts` — the ENF-01 acceptance test (criterion 4). Simulates a compaction/subagent boundary: in one process/function-call, `discussHook` writes a GovernanceRecord; then ALL in-memory state is dropped (the test function returns; only disk survives); in a SECOND, fresh function call (no cached modules, no shared variables — just `readSelection` from the same temp projectRoot), the record is reloaded and `renderInjection(reloaded.selectionResult)` is compared byte-for-byte against the original `renderInjection(original.selectionResult)`. Assertion: the two fragment strings are IDENTICAL (no clock in the selection — Pitfall 4 — guarantees this). Also asserts `JSON.stringify(reloaded.selectionResult) === JSON.stringify(original.selectionResult)`. Uses `mkdtempSync` for projectRoot, `rmSync` in a finally. This is the test that proves governance state survives the long-running loop.
