export async function runAdapter(adapter: GateAdapter, request: GateRequest): Promise<GateResult> {
  const result = await adapter.evaluate(request);
  validateGateResult(result);  // throws on malformed — never reaches the return below
  return result;
}

Import:
- type { GateAdapter } from "./adapters.js"
- type { GateRequest, GateResult } from "./types.js"
- { validateGateResult } from "./validate-gate-result.js"

The function is 3 lines: await evaluate, validate, return. The validateGateResult call throws on malformed output (it has an asserts result is GateResult signature). If it throws, the return is never reached — the malformed result never reaches a consumer (ENF-02 hard-fail).

CRITICAL: do NOT wrap adapter.evaluate(request) in a try/catch. A thrown evaluate() must propagate as-is (test 8 pins this — "tool crashed" reaches the caller, not a validation-shaped error). Only the OUTPUT is validated; the call itself is the adapter's responsibility. A try/catch around evaluate would swallow adapter runtime errors into validation errors, hiding real failures.

This is the lazy-dev minimum: 3 lines, no abstraction, no error-massage. The wrapper's only job is the integrity gate between adapter output and consumer.
