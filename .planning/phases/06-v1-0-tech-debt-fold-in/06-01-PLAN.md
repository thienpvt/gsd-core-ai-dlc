<objective>
Close the concurrent-writer race (TD-03, WR-05) by extracting a single shared atomic-write helper that uses a unique temp suffix (PID + monotonic counter) before the atomic rename. Both call sites — `audit-artifact.ts:68 atomicWriteText` and `state-store.ts:62 atomicWriteJson` — delegate to it so the fixed `.tmp` suffix collision is eliminated.

Purpose: The v2 adapter surface (Phases 7-10) will write governance artifacts from concurrent gate hooks. The current `${finalPath}.tmp` suffix means two writers to the same `GOVERNANCE.md` race — the second `writeFileSync(tmp)` truncates the first's in-flight content, then both `renameSync` to the same final path. A unique temp per writer closes the race.

Output: `src/governance/atomic-write.ts` (shared helper), updated `audit-artifact.ts` + `state-store.ts` delegating to it, regression tests still green, new concurrent-write property test.
</objective>

|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Tampering | `atomicWriteText` / `atomicWriteJson` fixed `.tmp` suffix | high | mitigate | Replace fixed `.tmp` with unique temp suffix (PID + counter) via shared `atomicWriteFile` helper. Two concurrent writers no longer share a temp path, so neither truncates the other's in-flight content. The atomic `renameSync` then wins-last semantics on the final path, and each temp is complete. |
| T-06-02 | Information Disclosure | Orphaned temp files under repeated write failures | low | accept | Best-effort `rmSync(tmpPath, { force: true })` on rename failure; if even that throws, the temp persists but is not read as the record (readSelection reads the final path, never any temp).
