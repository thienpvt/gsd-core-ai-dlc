   (The `as const` makes it a readonly tuple — STUB_NAMES is the single source of truth; the Maps and the tests both derive from it.)

3. Export the noopAdapter factory:
   export function noopAdapter(name: string): GateAdapter {
     return {
       name,
       async evaluate(request: GateRequest): Promise<GateResult> {
         return {
           gateId: request.gateId,
           status: "pass",
           findings: [],
           evaluatedBy: name,
           evaluatedAt: new Date().toISOString(),
         };
       },
     };
   }
   (noop = "nothing to report" contract per Area 2 — pass with empty findings. evaluatedAt uses new Date().toISOString() which produces the TD-01 strict shape YYYY-MM-DDTHH:mm:ss.sssZ.)

4. Export the echoAdapter factory:
   export function echoAdapter(name: string): GateAdapter {
     return {
       name,
       async evaluate(request: GateRequest): Promise<GateResult> {
         return {
           gateId: request.gateId,
           status: request.rules.length > 0 ? "fail" : "pass",  // ponytail: echo marks non-empty rules as fail so a caller sees findings; Phase 8 real adapters set status from tool output. Upgrade path: real adapters replace echo entirely.
           findings: request.rules.map((rule) => ({
             id: rule.id,
             severity: rule.severity,
             message: rule.summary,  // echo uses the rule summary as the finding message (testability — mirrors the request verbatim)
           })),
           evaluatedBy: name,
           evaluatedAt: new Date().toISOString(),
         };
       },
     };
   }
   (echo mirrors request.rules as findings per Area 2 — for testability. The findings carry id + severity from the rule; message uses the rule.summary so the finding is self-describing. evidence is omitted (optional).)

5. Export the static ADAPTERS Map (noop variants) and ECHO_ADAPTERS Map (echo variants), both keyed by name, built from STUB_NAMES:
   export const ADAPTERS: ReadonlyMap<string, GateAdapter> = new Map(
     STUB_NAMES.map((name) => [name, noopAdapter(name)] as const),
   );
   export const ECHO_ADAPTERS: ReadonlyMap<string, GateAdapter> = new Map(
     STUB_NAMES.map((name) => [name, echoAdapter(name)] as const),
   );
   (ReadonlyMap — no dynamic loader, no mutation API. Per Area 2: static Map, no dynamic loader in Phase 7.)

The stubs produce schema-valid GateResult by construction (gateId from request, status in enum, findings array, evaluatedBy = name, evaluatedAt = ISO string). 07-04's runAdapter wrapper validates this at the boundary — stubs do not self-validate (single responsibility: the wrapper owns the integrity gate, per Area 3).
