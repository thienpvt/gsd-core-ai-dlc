<objective>
Build the declarative GSD capability manifest that registers governance at `discuss:pre` (and forward-declares `execute:pre`), and the discuss-gate implementation: a deterministic risk heuristic that widens domain subscription, a thin hook adapter that marshals loop context into the existing pure cores (`validateSignal` -> `select` -> `renderInjection`), and the atomic state-store that persists the full SelectionResult to `.planning/governance/` so it survives compaction and subagent boundaries.

This is Wave 1: nothing in this plan depends on later plans. It produces the manifest, the risk + state-store primitives, and the discuss hook. The execute hook (04-02) reads state this plan writes; the consent flow (04-03) unlocks the manifest for the live loop.

The manifest uses the EXACT field names verified against the installed `code-review` capability and `capability-validator.cjs` (RESEARCH §1) — `id`, `role`, `version`, `title`, `description`, `tier`, `requires`, `engines`, `runtimeCompat`, `skills`, `agents`, `hooks`, `config`, `steps`, `contributions`, `gates`; step = `{ point, ref: { skill }, produces, consumes, when, onError }`. The validator rejects a `ref.skill` that starts with `gsd-` or is absent from `skills[]` (capability-validator.cjs:1215-1230), so the stems are unprefixed and declared. `when: "governance.enabled"` is evaluated by the registry, NOT the hook (Pitfall 8).

Built test-first where there is business logic (risk heuristic, state-store atomic write, hook marshaling) per active TDD mode; the manifest and SKILL.md are declarative glue (type: execute).

Purpose: this plan makes the overlay discoverable by GSD's capability loader and attaches the `<governance>` fragment at the discuss gate, satisfying GATE-01 (criterion 2) and laying the persistence substrate for ENF-01 (criterion 4, completed in 04-02).
Output: the `aidlc-governance` capability manifest, the discuss SKILL.md, `src/governance/{paths,risk,state-store,discuss-hook}.ts` + tests.
</objective>

- `src/governance/paths.ts` — single-sourced path helpers: `governanceDir(projectRoot)`, `selectionStatePath(projectRoot)` (=> `<root>/.planning/governance/selection-state.json`), `phaseDir(projectRoot, phase)` (=> `<root>/.planning/governance/phase-<NN>/`), `phaseRecordPath(projectRoot, phase, key)`. Centralizing these here prevents the path drift the prompt flags as a pitfall.
- `src/governance/risk.ts` — the deterministic risk heuristic (RESEARCH §4). Exports `classifyRisk(signal: TaskSignal, phase: Phase): RiskTier` where `RiskTier = "critical" | "elevated" | "baseline"`, plus `riskAdjustedDomains(tier: RiskTier, baseDomains: string[]): string[]`. Pure: keyword scan (auth/mfa/secret/credential/token/password/eval/payment/pci/card/gdpr/pii/crypto/injection) over `signal.keywords` + path scan (`auth/`, `payment/`, `security/`, `crypto/`) over `signal.paths` -> critical; else `phase === "construction"` && broad-change (paths.length >= 3) -> elevated; else baseline. Domains stack: critical => union(base, ["security", "payments"]); elevated => union(base, ["security"]); baseline => base unchanged. Deterministic (no clock, no random, no I/O).
- `src/governance/risk.test.ts` — RED-first node:test suite: each tier trigger, the path-only trigger, the construction-broad-change trigger, the baseline default, the domain-stacking for each tier, and a determinism assertion (repeated call equality). Plus a critical-auth-task case proving the security domain is subscribed (the recall lever — a baseline subscription would miss critical security rules).
- `src/governance/state-store.ts` — atomic read/write of `.planning/governance/` records. Exports `type GovernanceRecord = { phase: Phase; taskSignal: TaskSignal; selectionConfig: SelectionConfig; selectionResult: SelectionResult; riskTier: RiskTier; timestamp: string }` (timestamp on the WRAPPER only — Pitfall 4), `writeSelection(record, projectRoot): void` (atomic: `writeFileSync(tmp)` then `renameSync(tmp, final)` — survives a mid-write crash; RESEARCH §5), `readSelection(projectRoot): GovernanceRecord | null` (defensive parse — a malformed record throws a loud Error, never silently returns null/empty — Pitfall 7), and `writePhaseRecord / readPhaseRecord` for the per-phase dirs (same atomic pattern).
- `src/governance/state-store.test.ts` — RED-first node:test suite over a temp project root (mkdtempSync): write+read round-trip is byte-identical on `selectionResult` (the no-clock-in-selection guarantee); a `.tmp` left over from a simulated crash is NOT picked up as the record; a malformed `selection-state.json` (non-JSON, or JSON missing selectionResult) makes `readSelection` THROW (loud, not silent empty); concurrent-ish writes (two sequential writes) leave the LATER record intact (atomic rename).
- `src/governance/discuss-hook.ts` — the thin hook adapter. Exports `discussHook(args: { projectRoot: string; statePath?: string; indexPath?: string; taskSignal: TaskSignal; baseDomains?: string[]; budget?: number }): { fragment: string; record: GovernanceRecord }`. Reads `.planning/STATE.md` (or `statePath`) for the current phase (fail loud if missing/unparseable — Pitfall 7), reads the rule index from `indexPath` (default `rule-index.json`; fail loud if missing — never silent empty fragment), calls `classifyRisk` -> `riskAdjustedDomains` to build `SelectionConfig`, calls `validateSignal` (loud on invalid), `select(index, signal, config)`, `renderInjection(result)`, and `writeSelection(record, projectRoot)`. Returns the fragment for the caller/SKILL to attach to the discuss context. The hook does NOT re-implement matching, rendering, or budget — `select`/`renderInjection` are the sole logic sources (Pitfall 3).
- `src/governance/discuss-hook.test.ts` — RED-first node:test suite over a temp project root with a fixture index (reuse the existing `test/fixtures/eval/` or a tiny inline temp corpus): (a) given a baseline signal + construction phase, produces a `<governance>` fragment AND persists a record whose reloaded `selectionResult` equals the in-memory one; (b) given an auth-keyword signal, the persisted `selectionConfig.domains` includes "security" AND the selection includes security-domain rules a baseline subscription omits (the risk-widens-subscription recall lever); (c) given a missing/unreadable index path, the hook THROWS (never silently emits an empty fragment — Pitfall 7); (d) given an invalid TaskSignal, `validateSignal` rejects loud BEFORE select runs.
- `test/fixtures/governance-store/.gitignore` — ignores everything under the fixture store's `.planning/governance/` (the tests use mkdtempSync; this fixture marker keeps a real-corpus integration test from polluting the repo). The integration render-hooks test for criterion 1 is owned by 04-03 (post-consent); 04-01's acceptance is the loader discovering the manifest (`gsd-tools capability list`).
