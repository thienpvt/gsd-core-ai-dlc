<objective>
Build the summary injector — the first half of Phase 3's anti-bloat mechanism (SEL-02). A pure `renderInjection(result: SelectionResult): string` renders the selected rules' summaries into a single tagged `<governance>` markdown block, and a thin `governance inject` CLI wrapper reads a SelectionResult (stdin or `--input`) and writes the fragment to stdout. This is Wave 1, depending only on Phase 2's already-built `SelectionResult`.

The defining property is structural: the injector has NO file-read path at all. It reads only fields already present on the `SelectionResult` (`selected[].summary` is the frontmatter summary — the body was quarantined in `load.ts` and never reached the index per PACK-04), so summary-only injection (SEL-02) is true by construction, not merely by test. The fast-check no-body-canary property (ROADMAP success criterion 3) is the belt-and-suspenders proof on top of that guarantee, mirroring how Phase 1 proved PACK-04 both structurally and by property.

Built test-first per the active TDD mode: the fragment shape, the severity-then-id ordering, the empty-selection form, and the no-body property are RED-first against a stub `renderInjection`, then implemented GREEN. The CLI wrapper honors Phase 2's `budgetExceeded` signal — warning and exiting non-zero while still emitting the fragment — so an over-budget selection can never silently reach the working context (SEL-05 continuity).

Purpose: this plan produces the fragment Phase 4 will inject into the discuss/execute gates, proving "enough governance in-context to be safe, little enough to avoid bloat."
Output: the pure `renderInjection` + `SEVERITY_ORDINAL` in `src/inject/inject.ts`, its unit + no-body property suites, and the `governance inject` command wired into the CLI dispatch with a smoke test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SelectionResult (--input / stdin) -> governance inject -> renderInjection | Untrusted external JSON crosses into the injector at the CLI seam. A malformed payload must be rejected loudly, never silently coerced into an empty fragment (an empty governance block would silently drop ALL governance from the working context — an under-injection footgun). |
| selected[].summary -> <governance> fragment -> working context (Phase 4) | The fragment is what reaches the LLM working context. It must carry summaries + pointers only, never a rule body — the anti-bloat + no-leak guarantee. The injector's no-file-read structure is the primary control; the fast-check property is the proof. |
| result.budgetExceeded -> inject CLI -> exit code | The budget is the guard on "little enough to avoid context bloat". An over-budget selection must produce a loud signal (non-zero exit) and must never silently emit an over-budget fragment as if it were fine. |
