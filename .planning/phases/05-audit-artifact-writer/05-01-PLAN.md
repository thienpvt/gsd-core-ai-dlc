<feature>
  <name>Machine-derived governance audit record</name>
  <files>src/governance/audit-artifact.ts, src/governance/audit-artifact.test.ts</files>
  <behavior>
    - Given a GovernanceRecord with selected rules, buildAuditRecord returns rules_applied with id, severity, summary, matchedAxis, and matchedValue copied one-to-one and in order.
    - Given skipped rules with reasons out-of-phase, out-of-scope, out-of-scope-by-trigger, and superseded, buildAuditRecord returns rules_skipped with public audit reasons only and preserves selector_reason, detail, scope, and sourceFile when present.
    - Given selector reason out-of-scope, normalize to out-of-scope-by-trigger and preserve selector_reason as out-of-scope.
    - Given an unknown skipped reason, buildAuditRecord throws.
    - Given the same GovernanceRecord twice, renderGovernanceMarkdown parses to deeply equal rules_applied and rules_skipped both times.
    - Given missing selection-state.json, writeGovernanceAudit throws instead of emitting an empty audit.
    - Given malformed selection-state.json, writeGovernanceAudit propagates the state-store malformed-state error.
    - Given compiled direct runner arguments, `node dist/governance/audit-artifact.js <projectRoot> <outputPath>` writes `GOVERNANCE.md` only when `outputPath` basename is `GOVERNANCE.md` and resolved path stays under `<projectRoot>/.planning/phases/`.
    - audit-artifact.ts imports readSelection but imports no select core, validateSignal, classifyRisk, discussHook, or executeHook.
  </behavior>
  <implementation>Use TypeScript types from src/types.ts and GovernanceRecord/readSelection from src/governance/state-store.ts; render fenced JSON in Markdown with stable JSON.stringify indentation and no current clock; add an ESM-safe direct runner for the compiled `dist/governance/audit-artifact.js` entrypoint.</implementation>
</feature>
