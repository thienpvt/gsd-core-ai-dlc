<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Adapter -> verify hook | Adapter output is untrusted structured data. |
| Selection state -> verify request | Persisted selected rules become enforcement request input. |
| Verify hook -> evidence store | Adapter result becomes durable governance evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-09 | Tampering | adapter output | high | mitigate | Route all verify adapter output through `runAdapter`; tests inject spoofed/malformed output and assert rejection. |
| T-08-10 | Spoofing | adapter attribution | high | mitigate | `runAdapter` enforces `evaluatedBy === adapter.name`; tests include mismatched attribution. |
| T-08-11 | Repudiation | verify evidence | high | mitigate | Persist request/result/metadata with source `aidlc-governance-verify` and selected applied rules. |
| T-08-12 | Information Disclosure | verify request | low | accept | Evidence stores rule summaries already selected by governance; no rule bodies or scanner output are introduced. |
| T-08-SC | Tampering | package installs | low | accept | No package install task exists. |
</threat_model>
