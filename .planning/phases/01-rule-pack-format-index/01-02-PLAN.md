<objective>
Harden the frontmatter contract from the walking skeleton's minimal schema into the full PACK-01 authoring format and the PACK-03 advisory/binding classification, validated by Ajv (draft 2020-12) with fail-loud, field-named errors. This is the data contract every later phase inherits, so it is built test-first.

Scope-vs-directory (D-09) and duplicate-id (D-12) are programmatic checks owned by plan 01-03, not the schema.

Purpose: The schema is the single machine-checkable gate on rule authoring; a wrong schema silently breaks recall (D-03) or lets a binding rule ship with no contract (D-15).
Output: full frontmatter.schema.json + validate.ts error formatter + two table-driven test suites.
</objective>
